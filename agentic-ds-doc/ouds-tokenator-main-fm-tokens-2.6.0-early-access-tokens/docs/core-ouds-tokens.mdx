# OUDS Core Tokens

Core tokens are the **foundational layer** of the OUDS design system. They define the raw, resolved values — exact hex codes, pixel sizes, font names — that all semantic tokens reference. You should generally consume these indirectly through semantic tokens (`color.action.primary`, `space.inset.md`, etc.). Use core tokens directly only when no semantic token exists, or when building the token layer itself.

> **Convention** — CSS custom properties follow the pattern `--core-ouds-{category}-{subcategory}-{step}`. For example, `core.ouds.color.functional.gray.light.80` becomes `--core-ouds-color-functional-gray-light-80`.

---

## Typography

The typography tokens form a complete type scale covering font families, sizes, weights, line heights, and letter spacing. The numeric suffixes on scale tokens are **relative scale identifiers**, not pixel values — larger numbers always mean larger/heavier. The pixel value is the resolved output.

## Color

The color palette is split into two sub-groups: **functional** colors (fully opaque, named by hue) and **opacity** colors (semi-transparent variants for layering, overlays, and scrim effects). The numeric suffixes on functional palettes represent luminance steps within the hue — lower numbers are lighter, higher numbers are darker.
